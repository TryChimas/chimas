# deployment options ideas

## package git like

### $ chimas init
$ chimas init directory/
  copy default config *FROM PACKAGE* file to directory/chimas.conf
  **id est**, create a chimas 'repo'

$ chimas init --from-model chimas-model.toml.conf
  this creates a chimas repo using a model for BBS name, Boards list etc

```
[chimas]
name= "My Forum"

[boards]
my board 1
my board 2
another board
```

$ chimas init --admin-user "username" --admin-pass "pass"
  creates custom credentials for the administrator, if not, user is 'admin'
  and password is randomly generated and outputted from `init`

These also create a admin password.

### $ chimas start

$ chimas start
  if there is a chimas.conf in the current directory, loads it and starts
  if not, tell user this fact. "Error: Chimas.conf not found"

$ chimas start directory
  or
$ chimas start directory/
  if there is a chimas.conf in the current directory:
    change internal PATH to reflect that `directory/`
    loads it and starts
  if not, tell user this fact. "Error: chimas.conf not found"

WHEN STARTED, IT CREATES THE DB IF INEXISTET, LOADS THE CONFIG FILE AND RUNS NORMALLY.
